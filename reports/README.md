# Reports

This folder contains various audit and analysis reports for the Revolut Trader project.

## Contents

- **DOCUMENTATION_AUDIT_REPORT.md** — Comprehensive documentation audit (April 5, 2026)
  - Path reference fixes
  - Command documentation corrections
  - Project structure updates
  - GitHub workflows verification

## Purpose

This folder organizes all project audit reports, analysis results, and documentation reviews in one central location, keeping the project root clean and maintainable.

## Report Types

- **Documentation Audits** — Reviews of README, CLAUDE.md, AGENTS.md, and other docs
- **Code Quality Reports** — Output from static analysis and code review agents
- **Security Audits** — Security review findings and recommendations
- **Performance Analysis** — Profiling and optimization reports
- **Backtest Analysis** — Strategy performance reports (see also `revt-data/reports/` for runtime backtests)

## Adding Reports

When generating new reports from agent runs or manual audits:

1. Save reports in this folder with descriptive names
2. Include the date in the filename: `REPORT_NAME_YYYY-MM-DD.md`
3. Update this README with a brief description
4. Consider adding reports to `.gitignore` if they contain sensitive data

## Related Folders

- `revt-data/reports/` — Runtime trading analytics and backtest reports (generated by `revt db report`)
- `docs/` — Project documentation (guides, architecture, API reference)
